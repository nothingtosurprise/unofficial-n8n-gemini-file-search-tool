## Technical Notes

### n8n Convention Compliance

The scaffolding follows n8n community node package conventions:

1. **Directory Naming**: PascalCase for node directories (GeminiFileSearchStores, GeminiFileSearchDocuments)
2. **File Naming**:
   - Node files: `{NodeName}.node.ts` and `{NodeName}.node.json`
   - Credentials: `{CredentialName}.credentials.ts`
   - Operations: kebab-case or camelCase (to be determined in Phase 2)
3. **Package Structure**:
   - `dist/` directory for compiled output
   - `n8n` section in package.json with proper node and credential references
4. **Build Process**: TypeScript compilation with asset copying

### Development Workflow Support

**Scripts Available**:
- Development: `npm run dev` (watch mode)
- Building: `npm run build`
- Testing: `npm test` (all), `npm run test:unit`, `npm run test:integration`, `npm run test:e2e`
- Code Quality: `npm run lint`, `npm run format`
- Pre-publish: `npm run prepublishOnly` (automated checks)

### TypeScript Configuration Rationale

**Strict Mode**: Enabled for maximum type safety and early error detection
**ES2020 Target**: Modern JavaScript features while maintaining broad compatibility
**CommonJS Modules**: Required for n8n node compatibility
**Declaration Files**: Generated to support better IDE integration and type checking
