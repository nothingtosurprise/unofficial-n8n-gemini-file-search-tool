## Issues and Warnings

### Deprecation Warnings
During installation, the following deprecation warnings were encountered (non-blocking):

1. **inflight@1.0.6**: Module not supported and leaks memory (transitive dependency)
2. **@humanwhocodes/config-array@0.13.0**: Should use @eslint/config-array instead
3. **rimraf@3.0.2**: Versions prior to v4 are no longer supported
4. **glob@7.2.3**: Versions prior to v9 are no longer supported
5. **@humanwhocodes/object-schema@2.0.3**: Should use @eslint/object-schema instead
6. **eslint@8.57.1**: This version is no longer supported (but is the latest v8 release)

**Impact:** Low - These are transitive dependencies from other packages. Will be addressed in future updates.

### Security Vulnerabilities

**Critical Severity: 5 vulnerabilities detected**

**Vulnerability Details:**
- **Package**: form-data (versions 4.0.0 - 4.0.3)
- **Issue**: Uses unsafe random function for choosing boundary
- **Advisory**: GHSA-fjxv-7rqg-78g4
- **Affected**: n8n-core, n8n-workflow, @n8n/backend-common, @n8n/decorators

**Resolution:**
```bash
npm audit fix
```

**Note**: These vulnerabilities are in the n8n-core and n8n-workflow dependencies themselves. This is a known issue with the current n8n versions. The vulnerabilities should be addressed by n8n maintainers in future releases. For development purposes, this is acceptable as we're not deploying to production yet.
