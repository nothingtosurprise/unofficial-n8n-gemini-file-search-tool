# 6.3 Usage Guide: Gemini File Search AI Model

**Date**: 2025-11-30
**Phase**: 6 - AI Model Integration
**Status**: Complete

## Overview

This guide explains how to use Gemini File Search as an AI Model in n8n, enabling single API call RAG (Retrieval-Augmented Generation) queries.

---

## Prerequisites

### Requirements
- **n8n self-hosted** installation (v1.80+)
- **Gemini API key** with File Search access
- **File Search Store** with uploaded documents

### Why Self-Hosted?
The LangChain Code Node is only available on self-hosted n8n installations. n8n Cloud does not support custom code execution in LangChain nodes.

## Quick Start

### Step 1: Get Your Gemini API Key

1. Go to [Google AI Studio](https://aistudio.google.com/apikey)
2. Create or select a project
3. Generate an API key
4. Copy the key for later use

### Step 2: Create a File Search Store

Using the Gemini File Search Stores node:
1. Add the node to a workflow
2. Select **Create** operation
3. Set a display name (e.g., "My Knowledge Base")
4. Execute to create the store
5. Note the store name (e.g., `fileSearchStores/abc123`)

### Step 3: Upload Documents

Using the Gemini File Search Documents node:
1. Add the node to a workflow
2. Select **Upload** operation
3. Set the store name from Step 2
4. Connect binary data input (PDF, TXT, etc.)
5. Execute to upload documents

### Step 4: Import the Workflow Template

1. Download `gemini-file-search-ai-agent.json` from `docs/examples/`
2. In n8n: **Workflows** → **Import from File**
3. Select the downloaded file

### Step 5: Configure the Model

In the "Gemini File Search Model" node, update CONFIG:

```javascript
const CONFIG = {
  apiKey: 'YOUR_API_KEY_HERE',
  model: 'gemini-2.5-flash',
  storeNames: [
    'fileSearchStores/YOUR_STORE_NAME'  // From Step 2
  ],
  metadataFilter: '',
  temperature: 0.7,
  maxOutputTokens: 2048,
};
```

### Step 6: Test

1. Click "Test workflow" or "Execute Workflow"
2. Check the output in "Process Response" node
3. Verify citations are included in the response

## Configuration Options

### Model Selection

| Model | Speed | Quality | Best For |
|-------|-------|---------|----------|
| `gemini-2.5-flash` | Fast | Good | Most use cases |
| `gemini-2.5-pro` | Slower | Better | Complex queries |
| `gemini-2.5-flash-lite` | Fastest | Basic | Simple queries |

### Temperature

Controls randomness in responses:
- `0.0` - Deterministic, consistent answers
- `0.7` - Balanced (default)
- `1.0` - More creative, varied responses

### Max Output Tokens

Limits response length:
- `256` - Short answers
- `1024` - Medium responses
- `2048` - Detailed answers (default)
- `4096` - Very long responses

### Metadata Filter

Filter documents before search using AIP-160 syntax:

```javascript
// Filter by category
metadataFilter: 'category="api"'

// Filter by date
metadataFilter: 'year>2023'

// Combine filters
metadataFilter: 'category="docs" AND version>1.0'

// Contains search
metadataFilter: 'tags:"security"'
```

## Accessing Citations

### In Workflow Output

The "Process Response" node extracts citations:
```json
{
  "answer": "The authentication process involves...",
  "citations": [
    {
      "index": 1,
      "title": "Auth Guide",
      "text": "To authenticate, first obtain...",
      "store": "fileSearchStores/docs"
    }
  ],
  "citationCount": 1
}
```

### In Code Node

Access raw grounding metadata:
```javascript
const response = $input.first().json;

// All grounding chunks
const chunks = response.additional_kwargs?.groundingChunks;

// Citation mappings
const supports = response.additional_kwargs?.groundingSupports;

// Process each citation
for (const chunk of chunks || []) {
  console.log('Source:', chunk.retrievedContext?.title);
  console.log('Store:', chunk.retrievedContext?.fileSearchStore);
  console.log('Text:', chunk.retrievedContext?.text);
}
```

### Citation Confidence Scores

```javascript
const supports = response.additional_kwargs?.groundingSupports || [];

for (const support of supports) {
  console.log('Text segment:', support.segment?.text);
  console.log('Confidence:', support.confidenceScores?.[0]);
  console.log('Source indices:', support.groundingChunkIndices);
}
```

## Error Handling

### Common Errors

| Error | Cause | Solution |
|-------|-------|----------|
| `401 Unauthorized` | Invalid API key | Check API key is correct |
| `404 Not Found` | Invalid store name | Verify store exists |
| `400 Bad Request` | Invalid filter | Check metadataFilter syntax |
| `429 Rate Limited` | Too many requests | Add delay between calls |
| `Empty response` | No matching docs | Check store has relevant content |

### Error Handling in Workflow

Add error handling to your workflow:
```javascript
try {
  const response = await model._generate(messages);
  return response;
} catch (error) {
  return {
    error: true,
    message: error.message,
    code: error.code || 'UNKNOWN'
  };
}
```

## Performance Tips

### 1. Use Appropriate Model
- Use `gemini-2.5-flash` for most queries
- Reserve `gemini-2.5-pro` for complex analysis

### 2. Optimize Store Size
- Keep stores under 20GB for best performance
- Split large document sets into topic-based stores

### 3. Use Metadata Filters
- Filter reduces search space
- Faster responses with targeted queries

### 4. Set Reasonable Token Limits
- Lower `maxOutputTokens` for faster responses
- Higher for detailed analysis

### 5. Cache Common Queries
- Use n8n's caching for repeated questions
- Store responses in database for FAQ patterns

1. **Never hardcode in production** - Use environment variables
2. **Use n8n credentials** - Store in n8n's credential system
3. **Rotate regularly** - Update keys periodically

### Data Privacy

2. **Use metadata filters** - Limit exposure
3. **Monitor usage** - Track API calls and costs

## Troubleshooting

### "Model not responding"

1. Check API key validity
2. Verify network connectivity
3. Check n8n logs for errors

### "No citations returned"

1. Verify store has documents
2. Check documents are in ACTIVE state
3. Ensure query relates to document content

### "Slow responses"

1. Reduce `maxOutputTokens`
2. Use `gemini-2.5-flash` instead of pro
3. Add metadata filter to narrow search

### "Incorrect answers"

1. Check document content quality
2. Improve system prompt instructions
3. Use lower temperature for factual queries

    │

```

    │

    ▼
Response with Citations
```

- **Reduced costs**
- **Simpler architecture**
