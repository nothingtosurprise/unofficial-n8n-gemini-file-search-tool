## Overview

This report documents the n8n workflow template that demonstrates how to use the `GeminiFileSearchChatModel` with n8n's AI Agent for single-call RAG queries.

---

## Workflow Template

### File Location
`docs/examples/gemini-file-search-ai-agent.json`

### Import Instructions

1. Open n8n (self-hosted instance)
2. Go to **Workflows** → **Import from File**
3. Select `gemini-file-search-ai-agent.json`
4. Update configuration in the LangChain Code Node
5. Test the workflow

## Node Descriptions

### 1. Manual Trigger
- **Type**: `n8n-nodes-base.manualTrigger`
- **Purpose**: Start workflow execution for testing

### 2. Set Query
- **Type**: `n8n-nodes-base.set`
- **Purpose**: Define the query to send to the AI Agent
- **Configuration**:
  ```json
  {
    "query": "What are the main topics covered in the documents?"
  }
  ```

### 3. Gemini File Search Model
- **Type**: `@n8n/n8n-nodes-langchain.code`
- **Mode**: `defineChatModel`
- **Purpose**: Custom LangChain model with File Search
- **Connection**: AI Agent's "Model" input

### 4. AI Agent
- **Type**: `@n8n/n8n-nodes-langchain.agent`
- **Purpose**: Orchestrate conversation with the model
- **Configuration**:
  - Prompt Type: `define`
  - System Message: Configurable assistant instructions

### 5. Process Response
- **Type**: `n8n-nodes-base.code`
- **Purpose**: Extract and format grounding metadata
- **Output**:
  ```json
  {
    "answer": "Response text...",
    "citations": [
      {
        "index": 1,
        "title": "Document Name",
        "text": "Relevant excerpt...",
        "store": "fileSearchStores/store-name"
      }
    ],
    "citationCount": 3,
    "tokenUsage": {
      "prompt": 100,
      "completion": 200,
      "total": 300
    }
  }
  ```

## Configuration Required

Before using the workflow, update the CONFIG object in the LangChain Code Node:

```javascript
const CONFIG = {
  // Your Gemini API key
  apiKey: 'YOUR_GEMINI_API_KEY',

  // Model selection
  model: 'gemini-2.5-flash',

  // Your File Search Store names (REQUIRED)
  storeNames: [
    'fileSearchStores/your-store-name'
  ],

  // Optional metadata filter
  metadataFilter: '',

  // Generation settings
  temperature: 0.7,
  maxOutputTokens: 2048,
};
```

## Customization

### Change the Query
Modify the "Set Query" node to change the question:
```json
{
  "query": "Your custom question here"
}
```

### Add System Instructions
Modify the AI Agent's System Message:
```
You are a helpful assistant that answers questions based on the provided documents.
Always cite your sources when answering.
Be concise and accurate.
```

### Multiple Stores
Add multiple stores to search across:
```javascript
storeNames: [
  'fileSearchStores/docs-store',
  'fileSearchStores/api-store',
  'fileSearchStores/guides-store'
],
```

### Metadata Filtering
Filter documents before search:
```javascript
metadataFilter: 'category="api" AND version>2.0',
```

## Response Processing

The "Process Response" node extracts key information:

```javascript
// Extract grounding metadata from AI Agent response
const response = $input.first().json;

// Get the output text
const outputText = response.output || '';

// Get grounding chunks (source citations)
const groundingChunks = response.additional_kwargs?.groundingChunks || [];

// Format citations
const citations = groundingChunks.map((chunk, index) => ({
  index: index + 1,
  title: chunk.retrievedContext?.title || 'Unknown',
  text: chunk.retrievedContext?.text?.substring(0, 200) + '...' || '',
  store: chunk.retrievedContext?.fileSearchStore || 'Unknown'
}));

// Get token usage
const tokenUsage = response.additional_kwargs?.usageMetadata || {};

return {
  answer: outputText,
  citations: citations,
  citationCount: citations.length,
  tokenUsage: {
    prompt: tokenUsage.promptTokenCount || 0,
    completion: tokenUsage.candidatesTokenCount || 0,
    total: tokenUsage.totalTokenCount || 0
  }
};
```

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| "API key not valid" | Invalid or missing API key | Check CONFIG.apiKey |
| "Store not found" | Invalid store name | Verify store exists in Gemini |
| "Empty response" | No matching documents | Check store has documents |
| "Connection refused" | n8n can't reach Gemini | Check network/firewall |
