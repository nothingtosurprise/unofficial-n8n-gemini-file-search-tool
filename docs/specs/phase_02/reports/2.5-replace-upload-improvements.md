# 2.5 Replace Upload Improvements - Feature Specification

**Date:** 2025-11-25
**Phase:** 2 - Core Implementation (Enhancement)
**Status:** Approved for Implementation
**Author:** Claude Code
**Version:** 1.1

---

## 1. Overview

This specification documents improvements to the `replaceUpload` operation in the Gemini File Search Documents n8n node. The enhancements provide a more intuitive user experience with intelligent defaults, metadata preservation capabilities, and flexible document matching options.

### 1.1 Goals

1. Make `oldDocumentFilename` optional with intelligent fallback to `displayName`
2. Make `displayName` required for Replace Upload operations
3. Add metadata preservation capability from the old document
4. **Add metadata key matching option** - Match and delete documents by metadata value
5. **Change operation order** - Delete BEFORE upload to prevent accidental deletion of new document
6. Improve overall user experience with clearer field descriptions

### 1.2 Non-Goals

- Changing the underlying Gemini API behavior
- Adding batch replace functionality (future phase)
- Supporting partial file updates (API limitation)
- Replacing the regular Upload operation (see 1.4)

### 1.4 Why Keep Regular Upload Operation?

The regular **Upload** operation remains available and is **recommended for simple uploads** without replacement logic. Here's why:

| Operation | Use Case | Performance |
|-----------|----------|-------------|
| **Upload** | New documents, no need to check/delete existing | **Fast** - Single API call |
| **Replace Upload** | Replace existing documents by name/metadata | **Slower** - Lists all docs, then deletes, then uploads |

**Replace Upload with `matchBy = 'none'`** behaves similarly to regular Upload but:
- Still requires `displayName` (while Upload makes it optional)
- Is designed for the replace workflow even when skipping deletion

**Recommendation:** Use regular **Upload** when you're certain the document doesn't exist or don't care about duplicates. Use **Replace Upload** when you need to find and delete old versions first.

### 1.3 Critical Design Decision: Delete Before Upload

**Why delete BEFORE upload instead of after?**

When matching by metadata key, multiple documents may match the criteria. If we upload first and then delete by metadata, we would accidentally delete the newly uploaded document (since it might have the same metadata value).

**Flow comparison:**

| Order | Risk |
|-------|------|
| Upload → Delete | New document may be deleted if it matches the search criteria |
| Delete → Upload | Safe - old documents removed before new one exists |

**Trade-off:** If the upload fails after deletion, the old documents are already gone. This is acceptable because:
1. The user explicitly chose to replace
2. Upload failures are rare
3. The binary data is still available for retry

### 2.2 Current Flow (PROBLEMATIC)

```
1. Upload new document with provided metadata
2. List all documents in store
3. Search for document matching `oldDocumentFilename` (case-insensitive)
4. If found: Delete old document  ← PROBLEM: Could delete new doc!
5. Return upload result + deletion info
```

### 2.3 Pain Points

1. **`oldDocumentFilename` is required** - Forces user to specify even when replacing with same name
2. **`displayName` is optional** - But it's the natural identifier for replacement
3. **No metadata preservation** - User must re-enter all metadata manually
4. **No metadata matching** - Cannot find documents by metadata value
5. **Wrong order** - Deleting after upload risks deleting the new document
6. **Single match only** - Cannot delete multiple old versions at once

## 4. Logic Flow

### 4.1 High-Level Flow (NEW - Delete Before Upload)

```
1. Validate inputs (storeName, displayName required)
2. Get binary data and validate file size
3. If matchBy != 'none':
   a. List all documents in store
   b. Find matching document(s) based on matchBy option:
      - displayName: Match by displayName field (case-insensitive)
      - customFilename: Match by oldDocumentFilename (case-insensitive)
      - metadata: Match by metadata key-value pair (exact match)
   c. If preserveMetadata enabled AND match(es) found:
      - Extract metadata from FIRST matching document
      - Merge with new metadata based on strategy
   d. DELETE old document(s) FIRST
      - If deleteAllMatches (metadata only): delete all matches
      - Else: delete first match only
4. Upload new document with (potentially merged) metadata
5. Return comprehensive result
```

**When matchBy = 'none':** Skip steps 3a-3d, just upload the new document (behaves like regular upload but with required displayName).

### 4.2 Implementation Pseudocode

```typescript
type MatchByType = 'none' | 'displayName' | 'customFilename' | 'metadata';

async function replaceUpload(index: number): Promise<ReplaceUploadResult> {
  // 1. Get parameters
  const storeName = getNodeParameter('storeName', index);
  const displayName = getNodeParameter('displayName', index); // Required
  const matchBy = getNodeParameter('matchBy', index, 'none') as MatchByType;
  const oldDocumentFilename = getNodeParameter('oldDocumentFilename', index, '');
  const matchMetadataKey = getNodeParameter('matchMetadataKey', index, '');
  const matchMetadataValue = getNodeParameter('matchMetadataValue', index, '');
  const deleteAllMatches = getNodeParameter('deleteAllMatches', index, false);
  const preserveMetadata = getNodeParameter('preserveMetadata', index, false);
  const metadataMergeStrategy = getNodeParameter('metadataMergeStrategy', index, 'preferNew');
  const forceDelete = getNodeParameter('forceDelete', index, true);
  const waitForCompletion = getNodeParameter('waitForCompletion', index, true);
  const customMetadataParam = getNodeParameter('customMetadata', index, {});
  const chunkingOptions = getNodeParameter('chunkingOptions', index, {});

  // 2. Validate required fields
  validateStoreName(storeName);
  validateDisplayName(displayName);

  // Validate conditional required fields
  if (matchBy === 'customFilename' && !oldDocumentFilename) {
    throw new NodeOperationError(this.getNode(),
      'Old Document Filename is required when matching by custom filename');
  }
  if (matchBy === 'metadata') {
    if (!matchMetadataKey) {
      throw new NodeOperationError(this.getNode(),
        'Metadata Key is required when matching by metadata');
    }
    if (!matchMetadataValue) {
      throw new NodeOperationError(this.getNode(),
        'Metadata Value is required when matching by metadata');
    }
  }

  // 3. Get binary data and validate
  const binaryData = this.helpers.assertBinaryData(index, binaryPropertyName);
  const fileBuffer = await this.helpers.getBinaryDataBuffer(index, binaryPropertyName);
  validateFileSize(fileBuffer.length);

  // 4. Initialize results
  let matchingDocuments: Document[] = [];
  let deletedDocuments: DeletedDocumentDetail[] = [];
  let firstMatch: Document | undefined;

  // 5. Build base metadata
  let finalMetadata = buildNewMetadata(customMetadataParam, chunkingOptions);
  finalMetadata.displayName = displayName;

  // 6. Only search and delete if matchBy != 'none'
  if (matchBy !== 'none') {
    // List documents and find matching ones
    const documents = await geminiApiRequestAllItems(
      'documents',
      'GET',
      `/${storeName}/documents`
    );

    matchingDocuments = findMatchingDocuments(
      documents,
      matchBy,
      displayName,
      oldDocumentFilename,
      matchMetadataKey,
      matchMetadataValue
    );

    firstMatch = matchingDocuments[0];

    // Preserve metadata from FIRST match if enabled
    if (preserveMetadata && firstMatch?.customMetadata) {
      finalMetadata.customMetadata = mergeMetadata(
        firstMatch.customMetadata,
        finalMetadata.customMetadata || [],
        metadataMergeStrategy
      );
    }

    // Validate merged metadata count
    if (finalMetadata.customMetadata?.length > 20) {
      throw new NodeOperationError(this.getNode(),
        `Merged metadata exceeds maximum of 20 items (got ${finalMetadata.customMetadata.length})`);
    }

    // DELETE old document(s) FIRST (before upload!)
    const documentsToDelete = deleteAllMatches ? matchingDocuments :
      (firstMatch ? [firstMatch] : []);

    for (const doc of documentsToDelete) {
      try {
        const qs = forceDelete ? { force: true } : {};
        await geminiApiRequest('DELETE', `/${doc.name}`, {}, qs);
        deletedDocuments.push({
          name: doc.name,
          displayName: doc.displayName,
          deleted: true,
        });
      } catch (error) {
        deletedDocuments.push({
          name: doc.name,
          displayName: doc.displayName,
          deleted: false,
          error: error.message,
        });
      }
    }
  }

  // 7. Upload new document
  let uploadResult = await geminiResumableUpload(
    storeName,
    fileBuffer,
    binaryData.mimeType,
    finalMetadata
  );

  if (waitForCompletion) {
    uploadResult = await pollOperation(uploadResult.name);
  }

  // 8. Return comprehensive result
  return {
    upload: uploadResult,
    deletedDocuments: matchBy !== 'none' ? {
      matchBy,
      matchCriteria: getMatchCriteria(matchBy, displayName, oldDocumentFilename,
        matchMetadataKey, matchMetadataValue),
      totalFound: matchingDocuments.length,
      totalDeleted: deletedDocuments.filter(d => d.deleted).length,
      deleteAllMatches,
      documents: deletedDocuments,
    } : undefined,
    metadata: preserveMetadata && matchBy !== 'none' ? {
      preserved: !!firstMatch?.customMetadata,
      sourceDocument: firstMatch?.name,
      strategy: metadataMergeStrategy,
      oldMetadataCount: firstMatch?.customMetadata?.length || 0,
      newMetadataCount: customMetadataParam.metadataValues?.length || 0,
      finalMetadataCount: finalMetadata.customMetadata?.length || 0,
    } : undefined,
  };
}
```

### 4.3 Find Matching Documents Function

```typescript
type MatchByType = 'none' | 'displayName' | 'customFilename' | 'metadata';

function findMatchingDocuments(
  documents: Document[],
  matchBy: MatchByType,
  displayName: string,
  oldDocumentFilename: string,
  matchMetadataKey: string,
  matchMetadataValue: string
): Document[] {
  switch (matchBy) {
    case 'displayName':
      // Match by displayName (case-insensitive)
      const displayNameLower = displayName.toLowerCase();
      return documents.filter(
        doc => doc.displayName?.toLowerCase() === displayNameLower
      );

    case 'customFilename':
      // Match by oldDocumentFilename (case-insensitive)
      const filenameLower = oldDocumentFilename.toLowerCase();
      return documents.filter(
        doc => doc.displayName?.toLowerCase() === filenameLower
      );

    case 'metadata':
      // Match by metadata key-value (exact match, case-sensitive)
      return documents.filter(doc => {
        if (!doc.customMetadata) return false;
        return doc.customMetadata.some(meta => {
          if (meta.key !== matchMetadataKey) return false;
          // Check all value types
          if (meta.stringValue !== undefined) {
            return meta.stringValue === matchMetadataValue;
          }
          if (meta.numericValue !== undefined) {
            return meta.numericValue.toString() === matchMetadataValue;
          }
          if (meta.stringListValue?.values) {
            return meta.stringListValue.values.includes(matchMetadataValue);
          }
          return false;
        });
      });

    default:
      return [];
  }
}
```

### 4.4 Metadata Merge Function

```typescript
function mergeMetadata(
  oldMetadata: CustomMetadata[],
  newMetadata: CustomMetadata[],
  strategy: 'preferNew' | 'preferOld' | 'mergeAll' | 'replaceEntirely'
): CustomMetadata[] {
  if (strategy === 'replaceEntirely') {
    return oldMetadata;
  }

  const oldMap = new Map(oldMetadata.map(m => [m.key, m]));
  const newMap = new Map(newMetadata.map(m => [m.key, m]));

  if (strategy === 'preferNew') {
    // Start with old, override with new
    const merged = new Map(oldMap);
    newMap.forEach((value, key) => merged.set(key, value));
    return Array.from(merged.values());
  }

  if (strategy === 'preferOld') {
    // Start with new, override with old
    const merged = new Map(newMap);
    oldMap.forEach((value, key) => merged.set(key, value));
    return Array.from(merged.values());
  }

  if (strategy === 'mergeAll') {
    // Combine all (preferNew for conflicts)
    const merged = new Map(oldMap);
    newMap.forEach((value, key) => merged.set(key, value));
    return Array.from(merged.values());
  }

  return newMetadata;
}
```

## 6. Edge Cases and Error Handling

### 6.1 Edge Cases

| Scenario | Behavior |
|----------|----------|
| `matchBy=displayName`, no match found | Upload succeeds, `deletedDocuments.totalFound = 0` |
| `matchBy=metadata`, multiple matches, `deleteAllMatches=false` | Delete first match only |
| `matchBy=metadata`, multiple matches, `deleteAllMatches=true` | Delete all matches |
| `preserveMetadata=true`, multiple matches | Use metadata from FIRST match only |
| No matching document + `preserveMetadata=true` | Use only new metadata |
| Metadata match with numeric value | Convert to string for comparison |
| Metadata match with stringListValue | Check if value is in the list |
| Delete fails for some documents | Continue with others, report partial success |
| All deletes fail | Still upload new document, report failures |
| Upload fails after successful deletes | Old documents already deleted (acceptable trade-off) |

### 6.2 Error Handling

```typescript
// Required field validation
```

## 7. User Experience

### 7.1 Common Use Cases

| Use Case | Configuration |
|----------|---------------|
| **Upload only (no deletion)** | `matchBy` = "none" (or use regular Upload operation for better performance) |
| **Simple replacement (same name)** | `matchBy` = "displayName", `displayName` = "report.pdf" |
| **Rename during replacement** | `matchBy` = "customFilename", `displayName` = "report-v2.pdf", `oldDocumentFilename` = "report-v1.pdf" |
| **Replace by document ID** | `matchBy` = "metadata", `matchMetadataKey` = "documentId", `matchMetadataValue` = "doc-123" |
| **Replace all versions** | `matchBy` = "metadata", `matchMetadataKey` = "version", `matchMetadataValue` = "draft", `deleteAllMatches` = true |
| **Keep all metadata** | `preserveMetadata` = true, `metadataMergeStrategy` = "replaceEntirely" |
| **Update some metadata** | `preserveMetadata` = true, `metadataMergeStrategy` = "preferNew", provide new metadata |

**Performance Note:** For simple uploads without replacement, use the regular **Upload** operation instead of Replace Upload with `matchBy = "none"`. The Upload operation is faster as it doesn't need to list existing documents.

### 7.2 Field Order in UI

```
1. Store                          (required)
2. Input Binary Field             (required)
3. Display Name                   (required)
4. --- Match Options ---
5. Match Old Document By          (options: displayName, customFilename, metadata)
6. Old Document Filename          (only if matchBy = customFilename)
7. Metadata Key                   (only if matchBy = metadata)
8. Metadata Value                 (only if matchBy = metadata)
9. Delete All Matches             (only if matchBy = metadata)
10. --- Metadata Options ---
11. Preserve Old Document Metadata (checkbox)
12. Metadata Merge Strategy       (only if preserve=true)
13. Custom Metadata               (optional collection)
14. --- Advanced Options ---
15. Chunking Options              (optional collection)
16. Force Delete                  (default true)
17. Wait for Completion           (default true)
```

### 7.3 Field Descriptions

| Field | Description |
|-------|-------------|
| `displayName` | Display name for the new document. When matching by "Display Name", this is also used to find old documents. |
| `matchBy` | How to find old document(s) to delete before uploading. "Metadata" allows matching multiple documents. |
| `oldDocumentFilename` | The display name of the old document (when using custom filename matching). |
| `matchMetadataKey` | The metadata key to search for (e.g., "documentId", "version", "category"). |
| `matchMetadataValue` | The exact value to match. For numeric metadata, enter the number as text. |
| `deleteAllMatches` | When enabled with metadata matching, deletes ALL documents with matching metadata. Use with caution! |
| `preserveMetadata` | Copy metadata from the old document. If multiple docs deleted, uses metadata from the first match. |
| `metadataMergeStrategy` | How to combine old and new metadata. "Prefer New" is recommended for most cases. |
