      Internal

      Performs a resumable upload of a file to a Gemini file search store

      This function implements Google's resumable upload protocol with two steps:

      1. Start upload session and receive upload URL
      2. Upload file data to the session URL

      Supports files up to 100MB with custom metadata and chunking configuration.

      Parameters

      • this: IExecuteFunctions

        n8n execution context (IExecuteFunctions)

      • storeName: string

        Full store name (e.g., 'fileSearchStores/my-store-id')

      • file: Buffer

        File content as Buffer

      • mimeType: string

        MIME type of the file (e.g., 'application/pdf', 'text/plain')

      • metadata: IDataObject

        Document metadata including displayName, customMetadata, and chunkingConfig

      Returns Promise<any>

      Promise resolving to Operation object tracking the upload status

      Throws

      When upload session creation or file upload fails

      Throws

      When file size exceeds 100MB limit (validated before calling)

      Example

      // Upload a PDF with metadata
      const operation = await geminiResumableUpload.call(
        this,
        'fileSearchStores/my-store',
        fileBuffer,
        'application/pdf',
        {
          displayName: 'Technical Documentation',
          customMetadata: [
            { key: 'category', stringValue: 'docs' },
            { key: 'version', numericValue: 2.0 }
          ],
          chunkingConfig: {
            whiteSpaceConfig: {
              maxTokensPerChunk: 800,
              maxOverlapTokens: 100
            }
          }
        }
      );
      console.log(operation.name); // Use for polling operation status

      Return type is 'any' as response structure is defined by Google API
